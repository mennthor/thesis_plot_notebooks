What is listed:
1) pDaq Run Number 
2) Date when run was acquired (typically run start) 
3) Run length in hours 
4) Subdetector Status 
  Arranged in the following order InIce/IceTop 
  '-' no information 
  'x' significant problem in the subdetector, short, or very few events 
  'IT' subdetector active - I=IceCube, T=IceTop 
  Note: 'IT' is just the subdetector status (active) and not a decision if it was indeed good 
5) IC79 InIce status 
   - full    (86 strings are operational) 
   - partial (one or more strings are missing) 
6) Good Run Decision and comment 
   The good run decision is made for IceCube InIce. The reason for labeling the run as bad ('FAULTY') is given in the comment. The comment also includes a list of missing string for partial runs if that information was available from the DB (DroppedDOMs Lists). 

========================== 
Further details on sub-detector status (3): 
IceCube InIce 
============= 
'-' no information (no monitoring) 
'x' less than 10min, less than 6000 events, or significant problem in the InIce detector 
IceTop 
====== 
'-' no information (no monitoring) 
'x' less than 30min, less than 1000 events, significant problem in the IceTop detector
